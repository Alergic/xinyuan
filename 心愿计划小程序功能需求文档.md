# 心愿计划小程序功能需求文档

## 1. 项目概述

**项目名称：心愿计划**

心愿计划是一款帮助用户记录想买但暂时不能立刻购买的物品，并通过价格追踪、存款计划、任务清单和时间目标，逐步实现购买目标的小程序。

产品核心不是单纯收藏商品，而是帮助用户完成三件事：

1. 记录想买的东西。
2. 判断什么时候买比较合适。
3. 通过存款和计划一步步接近目标。

项目初期以核心功能为主，保证使用流程简单、顺畅、便捷；同时在数据结构和功能设计上为后期价格趋势、智能推荐、预算管理、多物品分配等功能预留扩展空间。

------

## 2. 核心目标

### 2.1 用户目标

用户可以：

- 记录想买的物品。
- 维护物品当前价格和历史价格。
- 为物品设置购买目标、存款计划和 deadline。
- 记录每次存钱来源。
- 查看每个物品的达成进度。
- 判断哪些物品接近可购买状态。

### 2.2 产品目标

产品需要做到：

- 添加物品足够快。
- 存款记录足够方便。
- 价格更新足够轻量。
- 首页能快速看到最重要的目标。
- 数据结构清晰，便于后续扩展。

------

## 3. 用户角色

初期仅支持普通用户角色。

用户通过微信授权登录后，可以管理自己的心愿物品、存款记录、任务、分类和价格记录。

## 5. 功能需求

## 5.1 心愿物品管理

### 5.1.1 添加心愿物品

用户可以添加一个想购买的物品。

字段如下：

| 字段         | 是否必填 | 说明                         |
| ------------ | -------- | ---------------------------- |
| 物品名称     | 必填     | 用户自定义名称               |
| 当前价格     | 必填     | 当前看到的价格               |
| 目标价格     | 可选     | 用户希望或可以接受的入手价格 |
| 商品图片     | 可选     | 可上传图片                   |
| 商品链接     | 可选     | 商品购买链接                 |
| 分类         | 可选     | 默认“未分类”                 |
| 优先级       | 可选     | 高 / 中 / 低，默认中         |
| 描述         | 可选     | 购买理由、规格、备注等       |
| Deadline     | 可选     | 希望完成购买或决策的时间     |
| 目标存款比例 | 可选     | 默认 100%                    |
| 状态         | 系统生成 | 默认“计划中”                 |

### 5.1.2 当前价格与目标价格规则

当前价格为必填。

目标价格为可选，原因是部分商品可能永远无法达到理想低价，用户只需要记录“当前可接受价格”。

价格逻辑如下：

- 当前价格：用户最近一次看到并记录的商品价格。
- 目标价格：用户心理预期或希望入手的价格，可为空。
- 若目标价格为空，则存款目标默认以当前价格为准。
- 若目标价格不为空，则可以用目标价格作为存款目标或价格提醒条件。
- 用户每次更新当前价格时，系统自动生成一条价格记录。

### 5.1.3 心愿物品状态

物品状态包括：

| 状态   | 说明                                         |
| ------ | -------------------------------------------- |
| 计划中 | 已添加，但暂无明确行动                       |
| 存款中 | 已有存款记录或存款分配                       |
| 可购买 | 存款进度达到设定比例，或当前价格低于目标价格 |
| 已购买 | 用户已完成购买                               |
| 已放弃 | 用户决定不再购买                             |
| 暂缓   | 用户暂时搁置                                 |
| 已逾期 | deadline 已过但未完成                        |

状态可由系统自动更新，也允许用户手动调整。

## 5.3 价格记录功能

### 5.3.1 更新当前价格

用户可以在物品详情页更新当前价格。

每次更新时生成一条价格记录。

价格记录字段：

| 字段     | 是否必填 | 说明                       |
| -------- | -------- | -------------------------- |
| 物品 ID  | 必填     | 关联心愿物品               |
| 价格     | 必填     | 本次看到的价格             |
| 记录日期 | 必填     | 默认当前时间               |
| 平台     | 可选     | 淘宝、京东、拼多多、闲鱼等 |
| 商品链接 | 可选     | 对应价格来源               |
| 备注     | 可选     | 活动、优惠券、会员价等     |

### 5.3.2 历史价格与史低价

“史低价”不是单独录入字段，而是根据价格记录自动计算。

规则：

- 每次更新当前价格都会进入价格记录。
- 系统从所有价格记录中计算最低价。
- 最低价即为该物品当前已知史低价。
- 物品详情页显示当前价格、已知史低价、价格更新时间。

### 5.3.3 价格趋势图

初期可预留趋势图能力，但仅在价格记录超过 3 次后展示。

规则：

- 价格记录少于 3 条时，不展示趋势图。
- 价格记录达到 3 条及以上时，可展示简单折线趋势图。
- 初期趋势图只需要展示价格随时间变化。
- 后期可扩展为多平台价格对比图。

## 5.4 存款功能

存款功能分为两种类型：

1. 专项存款
2. 通用存款池

这样既支持用户为某个物品专门存钱，也支持用户先把省下来的钱放进一个总池，再按比例或手动分配给多个物品。

------

## 5.4.1 专项存款

专项存款是指“这笔钱专门为某个物品存的”。

使用场景：

- 用户明确知道这笔钱是为了某个物品。
- 例如：“今天少喝奶茶，给耳机存 30 元”。

专项存款字段：

| 字段    | 是否必填 | 说明                 |
| ------- | -------- | -------------------- |
| 物品 ID | 必填     | 存给哪个物品         |
| 金额    | 必填     | 存入金额             |
| 日期    | 必填     | 默认当前时间         |
| 备注    | 可选     | 说明如何省下         |
| 标签    | 可选     | 如少吃外卖、兼职收入 |
| 类型    | 系统标记 | 专项存款             |

专项存款会直接计入对应物品的已存金额。

------

## 5.4.2 通用存款池

通用存款池是用户暂时不指定用途的存款总额。

使用场景：

- 用户只是今天省下了钱，但还没想好分配给哪个物品。
- 用户每月固定拿出一笔预算，再慢慢分配。
- 用户希望多个目标共享同一个存钱池。

通用存款池记录字段：

| 字段 | 是否必填 | 说明                           |
| ---- | -------- | ------------------------------ |
| 金额 | 必填     | 存入通用池的金额               |
| 日期 | 必填     | 默认当前时间                   |
| 备注 | 可选     | 来源说明                       |
| 标签 | 可选     | 少吃外卖、兼职收入、工资结余等 |

通用存款池支持两种分配方式：

### 手动分配

用户选择一个或多个物品，手动输入分配金额。

示例：

通用池有 500 元，用户分配：

- 耳机 200 元
- 键盘 100 元
- 旅行基金 200 元

### 按比例分配

用户选择多个物品，并设置分配比例。

示例：

- 相机 50%
- 耳机 30%
- 课程 20%

系统根据当前可分配金额自动计算分配结果。

初期建议先实现手动分配；按比例分配作为可扩展功能预留。

------

## 5.4.3 存款标签

每条存款记录都可以添加标签。

默认标签：

- 少吃外卖
- 减少娱乐
- 兼职收入
- 工资结余
- 闲置出售
- 奖金红包
- 定期存入
- 其他

标签作用：

- 帮助用户理解钱是怎么省下来的。
- 后续可统计不同标签贡献金额。
- 给用户形成正反馈。

------

## 5.4.4 已存金额计算

每个物品的已存金额由两部分组成：

物品已存金额 = 专项存款总额 + 通用池已分配给该物品的金额

通用池未分配金额不计入任何单个物品进度。

通用池余额计算：

通用池余额 = 通用池存入总额 - 已分配总额

## 5.5 任务清单功能

每个心愿物品可以绑定任务清单。

任务用于辅助用户完成购买前的准备，而不仅仅是存钱。

任务字段：

| 字段     | 是否必填 | 说明                           |
| -------- | -------- | ------------------------------ |
| 物品 ID  | 必填     | 关联物品                       |
| 任务标题 | 必填     | 任务名称                       |
| 任务描述 | 可选     | 详细说明                       |
| 任务类型 | 可选     | 存钱、比价、调研、冷静期、行动 |
| 截止日期 | 可选     | 任务自己的 deadline            |
| 是否完成 | 必填     | 默认未完成                     |
| 完成时间 | 系统生成 | 完成时记录                     |

默认任务模板：

- 找一个替代方案

## 5.6 Deadline 与预计完成日

### 5.6.1 Deadline

Deadline 是用户设置的目标时间点。

Deadline 类型：

| 类型         | 说明                   |
| ------------ | ---------------------- |
| 购买截止日   | 希望在某天前买到       |
| 冷静期结束日 | 到某天再决定是否购买   |
| 活动日期     | 如双十一、生日、旅行前 |
| 存款目标日   | 希望某天前存够钱       |

提醒规则：

- 距离 7 天提醒
- 距离 3 天提醒
- 当天提醒
- 逾期提醒

到期后可操作：

- 延期
- 暂缓
- 放弃
- 标记已购买

### 5.6.2 预计完成日

预计完成日由系统根据当前存款速度自动估算。

计算逻辑：

1. 获取该物品最近一段时间的存款记录。
2. 计算平均每日存款金额。
3. 用剩余所需金额除以平均每日存款金额。
4. 得出预计还需要多少天。
5. 推算预计完成日。

基础公式：

预计完成天数 = 剩余所需金额 / 平均每日存款金额

预计完成日 = 当前日期 + 预计完成天数

初期计算建议：

- 只统计最近 30 天的专项存款和已分配通用存款。
- 如果最近 30 天没有存款记录，则显示“暂无足够数据”。
- 如果平均每日存款金额为 0，则不计算预计完成日。
- 如果已存金额已达到目标，则显示“已达成”。

后期可扩展：

- 按周存款节奏计算。
- 排除异常大额收入。
- 支持用户设置固定每周 / 每月存款计划。
- 结合通用存款池自动分配预测。

## 7. 核心业务规则

### 7.1 价格规则

1. 当前价格必填。
2. 目标价格可选。
3. 每次更新当前价格，生成一条价格记录。
4. 史低价由价格记录自动计算。
5. 价格记录达到 3 条及以上，可以展示价格趋势图。
6. 若目标价格存在，且当前价格小于或等于目标价格，系统提示“已达到目标价格”。

### 7.2 存款规则

1. 存款分为专项存款和通用存款池。
2. 专项存款直接计入某个物品。
3. 通用存款池未分配金额不计入任何单个物品。
4. 通用池分配后，计入对应物品已存金额。
5. 每条存款记录可以添加备注和标签。
6. 物品已存金额 = 专项存款 + 通用池分配金额。
7. 存款进度 = 物品已存金额 / 存款目标金额。

### 7.3 存款目标金额规则

存款目标金额优先级：

1. 如果用户设置了目标价格，则默认使用目标价格。
2. 如果没有目标价格，则使用当前价格。
3. 用户可以手动修改存款目标金额。
4. 如果当前价格变动，不应强制覆盖用户手动设置的存款目标金额。

### 7.4 可购买规则

物品进入“可购买”状态的条件：

满足以下任一条件即可：

1. 存款进度达到用户设置的目标存款比例。
2. 当前价格小于或等于目标价格。
3. 用户手动标记为可购买。

### 7.5 Deadline 规则

1. Deadline 可为空。
2. Deadline 到期但未购买，物品状态可变为已逾期。
3. 已逾期物品允许延期、暂缓、放弃或标记已购买。
4. 预计完成日由系统自动计算，不需要用户手动填写。

## 11. 交互设计建议

整体风格建议：

- 清爽
- 温和
- 有计划感
- 不制造购物焦虑
- 不过度刺激消费

首页重点突出：

- 快完成的目标
- 即将到期的目标
- 通用池余额
- 今日可执行操作

物品卡片重点显示：

- 当前价格
- 已知史低价
- 已存进度
- deadline
- 状态

添加心愿流程应尽量轻：

第一步只要求：

- 名称
- 当前价格

其他字段可后续补充。

这样可以降低记录成本。

## 12. 一句话产品定义

心愿计划是一款帮助用户记录想买的东西、追踪价格变化、管理存款进度，并在合适时间理性完成购买的小程序。
